# 🛒 E-Commerce SQL Analysis Project

## 📌 Overview
This project focuses on analyzing an e-commerce database using SQL.  
It includes a collection of queries designed to extract meaningful business insights such as revenue trends, customer behavior, product performance, and inventory status.

---

## 🗂️ Database Structure
The project is built on the following tables:

- **customers** – Stores customer details (name, email, city, etc.)
- **products** – Contains product information (name, price, stock, category_id)
- **categories** – Defines product categories
- **orders** – Stores order transactions (customer_id, product_id, quantity, total_amount, order_date)

---

## 📊 Key Analysis & Queries

### 🔹 Basic Metrics
- Total number of customers
- Total number of products
- Total number of orders
- Total revenue generated

### 🔹 Product & Category Insights
- Revenue per category
- Product count per category
- Price range analysis (min, max, average)
- Top 5 best-selling products

### 🔹 Customer Analysis
- Total spending per customer
- Order count per customer
- High-spending customers (above threshold)
- Repeat customers
- Customers with no orders
- Customer distribution by city

### 🔹 Sales & Revenue Analysis
- Monthly revenue trends
- Revenue by category and month
- Average order value

### 🔹 Inventory Analysis
- Low stock products (stock < 100)

### 🔹 Reporting
- Full order details report combining all tables

---

## 🧠 Skills Demonstrated
- SQL Joins (INNER JOIN, LEFT JOIN)
- Aggregations (SUM, COUNT, AVG, MAX, MIN)
- Grouping & Filtering (GROUP BY, HAVING)
- Date formatting & time-based analysis
- Data analysis for business insights
